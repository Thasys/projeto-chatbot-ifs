## Project Overview

**IFS Transparência Inteligente** — an AI-powered chatbot that accepts natural language questions in Portuguese and translates them into SQL queries against a MySQL data warehouse of public financial data from Instituto Federal de Sergipe (IFS). Uses CrewAI's sequential multi-agent pipeline.

## Commands

```bash

# CLI mode (terminal only, no Streamlit)
python main.py

# Tests

```

## Architecture

The system has three layers connected sequentially:

**1. Frontend — `app_v2.py` (Streamlit)**
Chat UI with sidebar for query guide, configuration, and CSV report downloads. Manages session state, rate limiting (2s between queries), input validation via `guardrails.py`, and audit logging via `audit_logger.py`.

**2. Orchestration — `crew_definition_v2.py` (CrewAI)**
Three agents run in sequence:
- **Data Detective** — Parses user intent (RANKING / TOTAL / SEARCH), identifies entities via fuzzy search, outputs structured JSON. Uses `_get_latest_data_year()` to set the default year to the most recent year with data.
- **SQL Architect** — Generates and executes optimized SQL using few-shot examples from `knowledge_base.py`
- **Analyst** — Translates results to Portuguese, formats currency (BRL), adds insights and a confidence score (0–100%)

Query results are cached with a 5-minute TTL. Agent delegation is disabled; agents communicate only via task output.

**3. Data Layer — `db_connection.py`, `tools.py`**
SQLAlchemy connection pool (5 min, 10 max overflow, 1h recycle) to MySQL. Custom CrewAI tools: `search_entity_fuzzy`, `search_sql_memory`, `execute_sql`, `export_csv`.

**ETL Pipeline — `etl/`**
Daily scheduler (`etl/main.py`) fetches from the IFS transparency portal API (`extractor.py`), transforms (`transformer_v2.py`), and loads into the data warehouse (`loader.py`). Logs go to `logs/etl.log`; reports to `logs/etl_relatorio_*.json`.

## Database Schema

```
dw_ifs_gastos
├── v_financas_geral (VIEW) — primary query surface
│   columns: data, valor, favorecido_nome, id_favorecido,
│            unidade_pagadora, id_ug, tipo_despesa,
│            historico_detalhado, id_programa, id_natureza
├── dim_favorecido    — companies / providers
├── dim_ug            — organizational units / campuses
├── dim_programa      — government programs
├── dim_natureza      — expense categories
└── chat_audit_log    — request logs with confidence scores (created by audit_logger.py)
```

Views are created on first run by `setup_views.py`. The audit table is auto-created by `app_v2.py` on startup.

## Key Implementation Notes

- **Agent output format**: Agents use JSON mode with fallback parsing. If JSON extraction fails, the crew retries before surfacing an error.
- **Fuzzy entity matching**: `search_entity_fuzzy` loads up to 50,000 rows from `dim_favorecido` into memory for fuzzy match — avoid calling it in tight loops.
- **Windows line endings**: `Dockerfile` runs `sed -i 's/\r//' entrypoint.sh` to handle CRLF from Windows checkouts.
- **Python target**: 3.12 (Dockerfile base image `python:3.12-slim`).
- **Test markers**: `@pytest.mark.unit`, `@pytest.mark.integration`, `@pytest.mark.slow` — configure in `pytest.ini`.
- **CI/CD**: GitHub Actions runs `tests.yml` (lint + unit tests on Python 3.11 & 3.13), `docker.yml` (build/push image), and `etl-daily.yml` (scheduled data refresh).
